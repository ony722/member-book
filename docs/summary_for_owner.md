# メンバーブック — 社長向け1枚サマリー

（Round1レビュー指摘F-O5対応: 専門用語抜きで「何ができて何ができないか」をまとめた資料）

## これは何か
紙のメンバーブック（会員名簿）をめくる感覚で、会員を1人1ページずつ紹介できるWebサイトです。参考にしたのは「AAAメンバーブック電子版」のようなページめくり型ビューアです。

## 今回できること（MVP）
- 表紙 → 目次 → 会員ページとページをめくって閲覧できる（クリック/ドラッグ/ボタン/矢印キー、どれでもOK）
- 会員は1人1ページ。名前・自己紹介文・リンク（SNSやサイトへのリンク）を掲載
- 目次から見たい会員のページへ一発ジャンプできる
- スマホ・タブレット・PCどれでも見やすく表示される
- 会員20〜100人規模を想定
- 費用はかからない（無料のホスティングサービスで公開）

## 今回できないこと（意図的に対象外）
- 会員の写真掲載（文字とリンクのみという最初のご要望どおり）
- 印刷・PDFダウンロード・SNSへのシェアボタン
- ログイン制限（今回は誰でも見られる公開サイト）
- 管理画面からのデータ編集（今回はファイルを直接編集する運用）

## 今の進め方
- 最初は「見本のダミーデータ」（実在しない仮の会員情報）でサイトを完成させます
- 実際の会員情報を載せるのは、ご本人の掲載同意の取り方や、やめたいと言われた時の対応方法を決めてからにします（プライバシーへの配慮のため）

## データの更新方法
- 基本は担当エンジニアがファイル（テキストの一覧）を編集して更新します
- エンジニアがいなくてもブラウザ上で直接編集する簡易な方法もご用意する予定です（詳しくは開発が進んだ段階でご案内します）

## 将来追加できそうなこと（今回は見送り）
- データ編集用の管理画面
- 会員写真の掲載
- 検索・絞り込み
- シェア・印刷・PDF機能
- パスワード付きの限定公開

## 現在のステータス
設計・レビューが完了し、これから実装に着手します。実装完了後、動作確認（テスト）をして問題がなければ公開できる状態にします。
