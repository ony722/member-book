# スコープ表

## IN SCOPE（MVP対象）

- [ ] 機能A: フリップブック（紙めくり）UIで会員ページを閲覧できる
- [ ] 機能B: 1会員＝1ページ＝テキスト（名前・肩書・自己紹介等）＋リンク（1件以上）
- [ ] 機能C: ページ送り操作（クリック/ドラッグでめくる、前へ/次へボタン、キーボード矢印、目次からのジャンプ）
- [ ] 機能D: 表紙ページ・目次（一覧）ページ
- [ ] 機能E: レスポンシブ対応（PC・スマホ・タブレット）
- [ ] 機能F: 会員データをJSON等の静的データファイルで管理（20〜100件のサンプルデータを同梱）
- [ ] 機能G: 全画面表示切り替え
- [ ] 機能H: 現在ページ番号表示・目次への戻り導線

## OUT OF SCOPE（今回やらない）

- 会員データの管理画面（ログインして追加・編集するCMS/バックエンド）
- 会員写真・画像掲載（今回は「文字とリンクからなる」という明示要件のため対象外）
- 印刷／PDFダウンロード機能（参照サイトにはあるが今回のMVPでは対象外）
- シェア（SNS共有）機能
- 認証・限定公開（パスワード保護等）
- 実際の会員本人データの収集・入力（サンプルデータのみ）
- 多言語対応

## DEFER（延期・将来検討）

Round1レビュー指摘F-O6（優先度・着手トリガーの記載がなかった）への対応として各項目に優先度・トリガーを付記する。

| 項目 | 優先度 | 着手トリガー |
|---|---|---|
| 会員データ編集用の簡易管理画面（ノーコード運用） | 中 | RISK-003（非エンジニアの運用負荷）が実際に問題化した場合、またはGitHub Web UI簡易手順でも運用が回らないと判明した場合 |
| 会員写真対応（レイアウト拡張） | 低 | 社長から明示的な要望があった場合 |
| 検索・絞り込み機能 | 低 | 会員数が100件を大きく超え、目次からの発見性が問題化した場合 |
| シェア（SNS共有）／印刷／PDF機能 | 中 | 会員紹介による会員獲得導線として事業的価値が認められ、社長から優先指示があった場合 |
| 限定公開（パスワード認証）への切り替え | 低 | 実データ投入時に「誰でも見られる公開サイト」の方針自体が見直された場合（RISK-005参照） |
