# E2Eシナリオ — メンバーブック

各シナリオは7項目（前提条件/入力データ/実行手順/期待結果/検証方法/タイムアウト/クリーンアップ）で記述する。実行ツール: Playwright。

---

### E2E-01: 標準的な閲覧フロー（正常系1）

1. **前提条件**: `npm run build && npm run preview` でローカル配信中。サンプル`members.json`（20件）ロード済み
2. **入力データ**: 標準サンプル会員データ20件
3. **実行手順**: ルートURLにアクセス → 表紙をクリックしてめくる → 目次ページで3人目の会員名をクリック → 「次へ」ボタンで1ページ送る → 会員ページ内のリンクをクリック
4. **期待結果**: 表紙→目次→指定会員ページへ遷移し、「次へ」で次の会員ページに切り替わり、リンククリックで新規タブが開く（元タブは維持）
5. **検証方法**: 各遷移後にPage要素のテキスト内容をアサート。新規タブは`context.waitForEvent('page')`で捕捉しURLを検証
6. **タイムアウト**: 30秒
7. **クリーンアップ**: 開いた新規タブをクローズ

### E2E-02: 最小構成での正常動作（正常系2）

1. **前提条件**: `members.json`を1件のみに差し替えたテストビルドを使用
2. **入力データ**: 会員1件（`links`は1件のみ、`bio`は10文字）
3. **実行手順**: ルートURLにアクセス → 表紙→目次→唯一の会員ページまで遷移
4. **期待結果**: 目次に会員名が1件だけ表示され、会員ページ到達後「次へ」ボタンが非活性になる（FR-SYS-005の例外条件）
5. **検証方法**: 「次へ」ボタンの`disabled`属性をアサート
6. **タイムアウト**: 15秒
7. **クリーンアップ**: 不要（読み取りのみ）

### E2E-03: 最大規模相当での正常動作（正常系3）

1. **前提条件**: `members.json`を100件に拡張したテストビルドを使用
2. **入力データ**: 会員100件（目次が複数ページに分割される想定）
3. **実行手順**: ルートURLにアクセス → 目次を複数ページめくって全目次ページを確認 → 100件目の会員ページへ目次から直接ジャンプ
4. **期待結果**: 目次が複数ページに分割され、全100件の会員名が過不足なく表示される。100件目の会員ページへのジャンプが`flip`完了イベントの発火をもって完了する（ACC-FR-SYS-007、Round1レビュー指摘F-C6対応で時間しきい値アサーションから変更）
5. **検証方法**: 目次全ページの会員名テキストを集計し100件と一致することを確認。ジャンプ操作から`flip`完了イベントまでを待機してアサートし、所要時間は参考値としてテストログに出力する（CIのアサーション対象にはしない）
6. **タイムアウト**: 30秒
7. **クリーンアップ**: 不要（読み取りのみ）

### E2E-04: `page-flip`初期化失敗時のフォールバック（異常系1・外部依存障害）

1. **前提条件**: SDD.md定義の`__forceInitErrorForTest`フラグをURLクエリパラメータ経由で有効化し、`page-flip`初期化失敗を決定論的に再現する（Round1レビュー指摘F-C1対応。Phase 6実装時にURLクエリ方式へ設計修正済み、本番ビルドと同一の`dist/`に対して再現可能）
2. **入力データ**: 標準サンプル会員データ20件
3. **実行手順**: `/?__forceInitErrorForTest=1`にアクセス（初期化失敗を注入した状態）
4. **期待結果**: コンソールにエラーログが出力され、フォールバックとして「次へ/前へ」ボタンのみのシンプルなページ送りUIが表示され、閲覧自体は継続できる（SDD第2章エラーハンドリングフロー準拠）
5. **検証方法**: コンソールエラーの捕捉、フォールバックUI要素の存在確認、「次へ」ボタンでのページ遷移動作確認
6. **タイムアウト**: 15秒
7. **クリーンアップ**: モックの解除

### E2E-05: 不正な会員データによるビルド失敗（異常系2）

1. **前提条件**: テスト用に`url`が`http(s)://`で始まらない不正レコードを含む`members.json`を用意
2. **入力データ**: `{ "id": "m999", "name": "テスト太郎", "bio": "", "links": [{ "label": "不正リンク", "url": "javascript:alert(1)" }] }`を含むデータ
3. **実行手順**: `npm run build`を実行
4. **期待結果**: ビルドが非ゼロ終了コードで失敗し、エラーメッセージに不正レコードの`id`とフィールド名が含まれる
5. **検証方法**: プロセスの終了コードが0でないことを確認、標準エラー出力に該当`id`文字列が含まれることを確認
6. **タイムアウト**: 30秒
7. **クリーンアップ**: テスト用不正データファイルの削除

### E2E-06: 境界値（bio空文字・links0件）での表示崩れなし（境界系）

1. **前提条件**: `bio`が空文字列、`links`が空配列の会員レコードを含む`members.json`を使用
2. **入力データ**: `{ "id": "m020", "name": "リンクなし花子", "bio": "", "links": [] }`
3. **実行手順**: 目次から該当会員ページへジャンプ
4. **期待結果**: 紹介文欄は空表示（エラー表示や「undefined」等の文字列が出ない）、リンク欄自体が非表示になり、レイアウト崩れ（要素の重なり・はみ出し）が発生しない
5. **検証方法**: 該当DOM要素のテキストが空文字であること、リンクコンテナ要素が存在しない(`display:none`または未描画)こと、`document.documentElement.scrollWidth <= window.innerWidth`をアサート
6. **タイムアウト**: 15秒
7. **クリーンアップ**: 不要（読み取りのみ）

### 実装時に発見・修正した実装バグ（テスト実行により検出）

E2E実行の過程で、`page-flip`ライブラリの`SizeType`列挙型が実行時バンドルにエクスポートされておらず、`size: SizeType.STRETCH`が`TypeError`を起こして常にフォールバックUIに切り替わっていた実装バグを検出した。`'stretch' as SizeType`への修正、および`page-flip`本体のCSS（`stf__item`等の表示制御に必須）の読み込み漏れを修正し、実際のページめくり描画が正しく行われることを確認した（詳細: `docs/risk_register.md` RISK-001、`research/research_v2.md`「Phase 6実装時の追加発見」）。この経緯は、E2Eテストが「動いているように見えるが実際には動いていない」実装バグを検出した実例である。
