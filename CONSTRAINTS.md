## 7. 機微情報混入時のインシデント対応手順（Round1レビュー指摘F-O1対応）

SRS.md第11章で「Git履歴からの削除とGitHub Pagesの再デプロイ手順をCONSTRAINTS.mdに明記する」と約束した内容の具体化。会員データに機微情報（住所・電話番号・本人が公開を望まない情報等）が誤ってコミットされた場合の対応手順。

| 手順 | 内容 |
|---|---|
| 1. 即時対応 | 発覚後直ちに`members.json`から該当情報を削除するコミットを作成する（これだけでは履歴に残るため不十分、以下続行） |
| 2. 公開停止判断 | 影響が大きい場合、社長の承認を得てGitHub Pagesの公開を一時停止する（リポジトリ設定でPages機能を無効化） |
| 3. 履歴書き換え | 社長の明示承認を得た上で、`git filter-repo`（推奨）またはBFG Repo-Cleanerを用いて該当コミットを履歴から除去する。除去後は必然的にforce pushが必要となるため、C-AI-005の例外条項を適用する |
| 4. 再デプロイ | 履歴書き換え後、GitHub Actionsで再ビルド・再デプロイし、GitHub PagesのCDNキャッシュが更新されたことを確認する |
| 5. 検索エンジン対応 | 既にインデックスされている可能性がある場合、Google Search Consoleの削除リクエストツールで該当URLの削除申請を行う |
| 6. 記録 | 対応内容をPROGRESS.mdおよびrisk_register.md（RISK-006）に記録する |

**注意**: 手順3（履歴書き換え・force push）は必ず社長の事前承認を得てから実行する。承認なしに実行することはC-AI-006（CONSTRAINTS無断変更禁止の精神）に反する重大操作として扱う。
