# HARNESS.md — 安全ハーネス仕様（メンバーブック）

## 前提

本プロジェクトは以下を**持たない**: 外部API呼び出し、本番DB操作、自動投稿（SNS/メール）、課金操作。
そのため黄金ルール#3（dry-runがない破壊的操作はRedにする）が想定する「破壊的操作」自体が存在しない。以下のH1〜H5は、静的サイトプロジェクトに適合させた縮小版として定義する。

## H1: DRY_RUN
- **対象**: 該当なし（外部API呼び出しが存在しないため）
- **代替措置**: ビルドプロセス自体を「安全な試行」とみなす。`npm run build`はローカル/CIでの副作用がなく（`dist/`への出力のみ）、本番反映は別ステップ（GitHub Actionsのデプロイジョブ）で行う

## H2: 承認ゲート
- **対象**: `main`ブランチへのpush（GitHub Pagesへの自動デプロイをトリガーする唯一の操作）
- **実装方法**: `main`への直接pushではなく、Pull Request経由のマージを必須とする（C-DEV-002）。マージが事実上の承認ゲートとして機能する

## H3: 冪等性
- **対象**: ビルドプロセス全体（CC-05）
- **実装方法**: 同一`members.json`から生成される`dist/`の内容は常に同一。ビルドに乱数・現在時刻依存のファイル名を含めない

## H4: キルスイッチ
- **対象**: ローカル開発サーバー（`npm run dev`）およびビルドプロセス
- **実装方法**: Ctrl+Cでグレースフル停止（Viteの標準動作に準拠）。CIのビルドはGitHub Actions側のジョブキャンセル機能を利用する

## H5: 監査ログ
- **対象**: `main`ブランチへの変更履歴
- **実装方法**: Gitのコミット履歴＋GitHub Actionsのデプロイ履歴をもって監査ログとする（専用の`audit.log`は本プロジェクト規模では不要と判断。理由: 個人情報を扱う機微操作が存在しないため）
